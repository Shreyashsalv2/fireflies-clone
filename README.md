# Fireflies Clone — Meeting Notes & Transcription Platform

A full-stack clone of [Fireflies.ai](https://fireflies.ai). Browse a library of meetings, view an
**interactive transcript** synced to a media player, read **AI-generated** summaries / action items /
topics, **search** across everything, and manage meetings with full **CRUD**.

Real speech-to-text is out of scope (per the assignment); transcripts are seeded or pasted/uploaded, and
summaries are generated by **Groq** with a deterministic mock fallback so the app works even without a key.

> **Live demo**
> - **App (Vercel):** https://frontend-nine-sage-spasdcz85x.vercel.app
> - **API + Swagger docs (Render):** https://fireflies-clone-2g5d.onrender.com/docs

---

## Features

**Core**
- **Meetings library** — cards with title, date, duration, participants, tags; search, date filter, sort by recency/title.
- **Interactive transcript** — speaker labels + timestamps; **click a line to seek**, playback **highlights the active line** and auto-scrolls, plus **search within the transcript** (highlight + prev/next).
- **Media player** — seek bar, play/pause, ±10s. Simulated timeline (audio is a placeholder, as allowed by the spec).
- **AI summary, action items & key topics** — generated from the transcript by Groq; clickable topics seek the player.
- **Full CRUD** — create a meeting by pasting or uploading a transcript (`.txt`/`.vtt`/`.json`), edit metadata, delete, and add/edit/complete/delete action items. Everything persists.

**Extras**
- Global search page across all meetings (title / participant / summary / transcript).
- Export a meeting's transcript + summary to **Markdown** or **plain text**.
- **Dark mode** (persisted), **collapsible sidebar** (desktop focus-mode + mobile slide-in drawer), toasts, profile/account menu.
- "Coming soon" placeholders for the live bot, integrations, team/sharing, and real auth (a default user is assumed).

## Architecture

Monorepo with two independent apps. The frontend is a client-rendered SPA (React Query) that talks to the
FastAPI backend over a REST API; the backend persists to SQLite and calls Groq for summaries.

```
┌──────────────────────┐        HTTPS / REST        ┌──────────────────────────┐
│  frontend/ (Next.js) │  ───────────────────────▶  │  backend/ (FastAPI)      │
│  • App Router SPA    │   NEXT_PUBLIC_API_URL      │  • REST API (/api/*)     │
│  • React Query       │  ◀───────────────────────  │  • SQLModel ↔ SQLite     │
│  • Tailwind v4       │        JSON                │  • Groq service + mock   │
└──────────────────────┘                            └──────────┬───────────────┘
                                                               │
                                                        ┌──────▼──────┐   ┌──────────┐
                                                        │  SQLite DB  │   │ Groq API │
                                                        └─────────────┘   └──────────┘
```

Repo layout:
```
├── frontend/            # Next.js app (app/, components/, lib/)
├── backend/             # FastAPI app (app/: models, schemas, crud, routers/, services/, seed)
├── render.yaml          # Render blueprint for the backend
└── README.md
```

## Database Schema

SQLite via SQLModel. All child rows **cascade-delete** with their meeting.

```
meetings ──1:*── participants
   │      ──1:*── transcript_segments   (speaker, start_time, end_time, text, order_index)
   │      ──1:1── summaries             (overview, generated_by: seeded|groq|mock)
   │      ──1:*── action_items          (text, assignee?, completed, due_date?, order_index)
   │      ──1:*── topics                (title, start_time?  → clickable chapter)
   └──────*:*── tags                    (via meeting_tags link table)
```

| Table | Key columns |
|---|---|
| `meetings` | id, title, description, meeting_date, duration_seconds, media_url, created_at, updated_at |
| `participants` | id, meeting_id →, name, email?, role? |
| `transcript_segments` | id, meeting_id →, speaker, start_time, end_time, text, order_index |
| `summaries` | id, meeting_id → (unique), overview, generated_by, created_at, updated_at |
| `action_items` | id, meeting_id →, text, assignee?, completed, due_date?, order_index, created_at |
| `topics` | id, meeting_id →, title, start_time?, order_index |
| `tags` / `meeting_tags` | id, name (unique) / (meeting_id, tag_id) |

## API Overview

Base path `/api`. Interactive Swagger docs are auto-generated at **`/docs`**.

| Method | Endpoint | Description |
|---|---|---|
| GET | `/meetings` | List; query: `search`, `participant`, `meeting_date`, `sort` (recent\|oldest\|title) |
| POST | `/meetings` | Create from `transcript_text` or `segments`; triggers AI summary |
| POST | `/meetings/upload` | Create from an uploaded transcript file (multipart) |
| GET | `/meetings/{id}` | Full meeting detail |
| PATCH | `/meetings/{id}` | Edit title / description / participants / tags |
| DELETE | `/meetings/{id}` | Delete a meeting |
| POST | `/meetings/{id}/generate-summary` | Regenerate summary / action items / topics |
| POST | `/meetings/{id}/action-items` | Add an action item |
| PATCH | `/action-items/{id}` | Edit / complete an action item |
| DELETE | `/action-items/{id}` | Delete an action item |
| GET | `/search?q=` | Global search across meetings |
| GET | `/meetings/{id}/export?format=md\|txt` | Export transcript + summary |
| GET | `/api/health` | Health check |

## Getting Started (local)

**Prerequisites:** Node.js 20.9+ and Python 3.11+.

### 1) Backend (`backend/`)
```bash
cd backend
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env               # optional: add your GROQ_API_KEY for live AI
python -m app.seed                 # seed 5 sample meetings
uvicorn app.main:app --reload --port 8000
```
API at http://localhost:8000 · docs at http://localhost:8000/docs.
(On startup the DB auto-seeds if empty, so the app is never blank.)

### 2) Frontend (`frontend/`)
```bash
cd frontend
npm install
npm run dev                        # http://localhost:3000
```
The frontend defaults to `http://localhost:8000/api`; override via `frontend/.env.local` if needed.

## Environment Variables

**Backend** (`backend/.env`)
| Var | Purpose | Default |
|---|---|---|
| `GROQ_API_KEY` | Groq key for live summaries. Blank → deterministic mock. | *(empty)* |
| `GROQ_MODEL` | Groq model id | `llama-3.3-70b-versatile` |
| `FRONTEND_URL` | Allowed CORS origin (the Vercel URL in prod) | `http://localhost:3000` |
| `DATABASE_URL` | SQLAlchemy URL | `sqlite:///./fireflies.db` |

**Frontend** (`frontend/.env.local`)
| Var | Purpose | Default |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | Backend base URL incl. `/api` | `http://localhost:8000/api` |

> Secrets live only in local `.env` files (gitignored) or the host's dashboard — never committed.

## Deployment

### Backend → Render
1. **New + → Blueprint**, connect this repo (Render reads `render.yaml`; root dir `backend`).
2. Set env vars in the dashboard: `GROQ_API_KEY` (your key) and `FRONTEND_URL` (the Vercel URL, added after step 2).
3. Deploy. Note the URL, e.g. `https://fireflies-clone-api.onrender.com`.

### Frontend → Vercel
1. **Import** this repo. Set **Root Directory = `frontend`**.
2. Add env var `NEXT_PUBLIC_API_URL = https://<your-render-app>.onrender.com/api`.
3. Deploy, then set the backend's `FRONTEND_URL` to the Vercel URL and redeploy the backend.

**Notes**
- CORS already allows `*.vercel.app` (regex) plus `FRONTEND_URL`.
- Render's free tier sleeps after ~15 min → the first request can take ~50s (cold start). An uptime pinger hitting `/api/health` keeps it warm.
- Render's disk is ephemeral, so SQLite resets on redeploy/cold-start — the app **re-seeds the 5 meetings automatically**, so the dashboard is never empty. Use Postgres (`DATABASE_URL`) for durable user data.

## Assumptions
- **Single default user** — no real authentication (a placeholder account menu is shown).
- **No real audio** — the media player is a simulated timeline driven by transcript timestamps.
- **AI provider is Groq (Llama)**; summaries fall back to a local mock if the key is absent or the API fails, so creation never breaks.
- Pasted transcripts without timestamps get **synthesized timings** so the player still works.
